Detailed Description

Initializes XTD's logging facility from property map.

This singleton object creates and configures Logger, Formatter and Appender objects from a (key,value) property map given to configure method.

Class registration

You can specify which class name of Appender and Formatter you want to instantiate with the .class keys of the property map.

By default, ConfLoader only knows about XTD's formatters and appenders classes.

If you want to use your own class types you must first register them to ConfLoader's factory with the following methods :

  • registerFormatter<T> :
    • Registers given class type to formatter factory.
  • registerAppender<T> :
    • Registers given class type to appender factory.

You may also use following pre-defined macros to statically register your classes :

  • XTD_CORE_LOG_REGISTER_FORMATTER :
    • Registers an Formatter to class type to global ConfLoader instance.
  • XTD_CORE_LOG_REGISTER_APPENDER
    • Registers an Appender class type to global ConfLoader instance.

Registered classes must implement the following function prototype :

bool MyClass::create(const string& p_name, const map<string,string>& p_props);

Configuration

Properties defined loggers, appenders and formatters.

Each kind of object are identified by a key build from a prefix and a name

  • log.logger.<name> : defines a Logger
  • log.appender.<name> defines an Appender
  • log.formatter.<name> : defines a Formatter

The value of these keys depends on the object type and is described below.

Logger definition

The defining key of a Logger must follow the following syntax :

log.logger.<name> = <level>[, <appender_name>]

Where :

  • <name>
    • Logger's name which will be accessible from
    • log::getRoot().get(<name>)
    • The special value root stands for the RootLogger itself.
  • <level>
    • The level threshold label to use for this logger
  • <appender_name>
    • A comma separated list of appender names to add on defined logger.
    • Each name must correspond to a valid appender definition (see below).
    • The list of appender names may be empty.

Appender definition

For each <appender_name> found in logger definitions, the following key must be defined :

log.appender.<appender_name>.class = <class_name>

Where :

  • <class_name>
    • An Appender class name registered to the ConfLoader object

In addition, the following key may be defined :

log.appender.<appender_name>.formatter = <formatter_name>

Where :

  • <formatter_name>
    • The name of the defined formatter to use with this appender

Other keys may be required depending of the requested appender class. Please refer class documentation for details.

Formatter definition

For each <formatter_name> found in appender definitions, the following key must be defined :

log.formatter.<formatter_name>.class = <class_name>

Where :

  • <class_name>
    • An Formatter class name registered to the ConfLoader object

Other keys may be required depending of the requested formatter class. Please refer class documentation for details.

Exemple

The following properties defines three loggers with difference log levels :

  • the RootLogger (named root)
  • a logger for module proxy
  • a logger for submodule proxy.sensible

The root logger send log records to two appenders, A1 and A2 and the other loggers send records only to A1.

A2 uses a formatter named F1 which tweaks how location informations are displayed and colors module name in red.

log.logger.root = err, A1, A2
log.logger.proxy = debug, A2
log.logger.proxy.sensible = notice, A2
log.appender.A1.class = SyslogAppender
log.appender.A1.identity = my-bin
log.appender.A1.facility = LOG_USER
log.appender.A1.options = LOG_PID
log.appender.A2.class = StreamAppender
log.appender.A2.fd = cerr
log.appneder.A2.formatter = F1
log.formatter.F1.class = ColoredFormatter
log.formatter.F1.layout.location = %(file):%(line)
log.formatter.F1.style.module.fgcolor = red

Note: All three loggers use the same A2 formatter instance.

If you want to use distinct appender or formatters objects, you must define each one of them.
