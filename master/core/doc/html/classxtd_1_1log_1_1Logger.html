Detailed Description

Handles log record and log level.

The Logger handles user logs by filtering those under configured level and propagating the others to its registered Appender objects.

Object methods

  • getLevel()
    • Get current logger level.
  • setLevel(level)
    • Set logger active log level.
  • addAppender(object)
    • Add an Appender to internal list.
  • clearAppenders()
    • Remove all registered Appender.
  • log(format, args...)
    • Trigger level::emerg log.

Variadic arguments format

Logger::log and all following logging shorthands allow to provide a format message with variadic template argument. Functionally, its equivalent to plain old C-printf argument but allows to pass std::string and user-defined objects. We internally rely on boost::format, please see http://www.boost.org/doc/libs/1_58_0/libs/format/doc/format.html for more details

string l_name = "my name";
Logger& l_obj = log::getRoot();
l_obj.log(level::err, "message %d variadic args : %s, %.2f", 2, l_name, 2.256);

Logging shorthands

  • emerg(format, args...)
    • Trigger level::emerg log.
  • alert(format, args...)
    • Trigger level::alert log.
  • crit(format, args...)
    • Trigger level::crit log.
  • err(format, args...)
    • Trigger level::err log.
  • warning(format, args...)
    • Trigger level::warning log.
  • notice(format, args...)
    • Trigger level::notice log.
  • info(format, args...)
    • Trigger level::info log.
  • debug(format, args...)

Logger& l_obj = log::getRoot();
l_obj.emerg("emergency message");
l_obj.alert("alert message");
l_obj.crit("critical message");
l_obj.err("error message");
l_obj.warning("warning message");
l_obj.notice("notice message");
l_obj.info("info message");
l_obj.debug("debug message");

Thread Safety

  • Concurrent calls to multiple objects : YES
  • Concurrent calls to single object : YES

The class uses a mutex-based locking that provides thread safeness for all read and write operations on internal Appender list. However record forwarding to each appender it not lock. The Appender object is responsible of its own thread safeness.
